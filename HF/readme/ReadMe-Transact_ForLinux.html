Important notes

  • Before starting the deployment or rollback procedure, read this entire ReadMe file. If you have questions, contact Support for assistance.
  • If the Transact server does not start or perform as expected after the deployment of this fix, follow the steps in the Remove this fix section to revert your changes. However, we recommend contacting Support first.

Install this fix

  1. Stop the Transact server if it is running.
  2. Copy the deliverables/Transact-{RELEASE_VERSION}.FIX{OOS_ID}_ForLinux.zip file inside the deliverables folder of the PDT tool.

    3. For example, the path looks similar to the following: PDT-TOOL/deliverables

    Note: Make sure that only the Transact-{RELEASE_VERSION}.FIX{OOS_ID}_ForLinux.zip file is there and remove all the other files and folders. Otherwise, the PDT tool installs all the fixes located inside the deliverables folder.

  3. Run Terminal as Superuser and navigate to the PDT tool location.
  4. Run command generic_installer.sh to install the fix.
  5. Start the Transact server and perform the test cycle:
    • Start the Ephesoft server.

      • Note: In a clustered environment, we recommend starting nodes only after ALL nodes have been upgraded.

    • Perform verifications as suggested in the Important notes section.

Remove this fix

  1. Stop the Transact server if it is running.
  2. Copy the deliverables/Transact-{RELEASE_VERSION}.FIX{OOS_ID}_ForLinux.zip file inside the deliverables folder of the PDT tool.

    3. For example, the path looks similar to the following: PDT-TOOL/deliverables

    Note: Make sure that only the Transact-{RELEASE_VERSION}.FIX{OOS_ID}_ForLinux.zip file is there and remove all the other files and folders. Otherwise, the PDT tool uninstalls all the fixes located inside the deliverables folder.

  3. Run Terminal as Superuser and navigate to the PDT tool location.
  4. Run command generic_uninstaller.sh to uninstall the fix.
  5. Start the Transact server.
